# 指标采集 - 动态设备发现功能说明

**更新日期**: 2026-03-18  
**功能**: 动态获取服务器磁盘设备和网卡接口列表

---

## 📋 问题背景

### 原问题

在创建任务配置指标采集时，磁盘设备和网卡接口的下拉框是**静态硬编码**的：

```vue
<!-- 旧代码 -->
<el-select v-model="config.disk.devices" multiple>
  <el-option label="sda" value="sda" />     <!-- 硬编码 -->
  <el-option label="sdb" value="sdb" />     <!-- 硬编码 -->
  <el-option label="nvme0n1" value="nvme0n1" /> <!-- 硬编码 -->
</el-select>
```

**问题**：
1. ❌ 不同服务器的设备名不同（服务器 A 可能是 sda，服务器 B 可能是 nvme0n1）
2. ❌ 网卡接口名因服务器而异（eth0/ens192/enp3s0 等）
3. ❌ 多服务器任务无法统一配置

---

## ✅ 解决方案

### 方案 1：后端动态发现（已实现）

通过 SSH 连接到服务器，实时获取设备和网卡列表。

#### 后端 API

**1. 获取单个服务器的磁盘列表**
```http
GET /api/v1/servers/discovery/{serverId}/disks
```

**响应示例**：
```json
{
  "code": 0,
  "message": "success",
  "data": [
    {"name": "sda", "label": "sda"},
    {"name": "sdb", "label": "sdb"},
    {"name": "nvme0n1", "label": "nvme0n1"}
  ]
}
```

**2. 获取单个服务器的网卡列表**
```http
GET /api/v1/servers/discovery/{serverId}/interfaces
```

**响应示例**：
```json
{
  "code": 0,
  "message": "success",
  "data": [
    {"name": "eth0", "label": "eth0"},
    {"name": "ens192", "label": "ens192"},
    {"name": "bond0", "label": "bond0"}
  ]
}
```

**3. 获取多服务器的资源汇总**
```http
POST /api/v1/servers/discovery/resources
Content-Type: application/json

{
  "serverIds": [1, 2, 3]
}
```

**响应示例**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "disks": [
      {"name": "sda", "label": "sda"},
      {"name": "nvme0n1", "label": "nvme0n1"}
    ],
    "interfaces": [
      {"name": "eth0", "label": "eth0"},
      {"name": "ens192", "label": "ens192"}
    ]
  }
}
```

#### 前端实现

**MetricCollectConfig 组件**接收 `serverIds` prop：

```vue
<MetricCollectConfig 
  v-model="formData.collectConfig" 
  :server-ids="selectedServerIds" 
/>
```

**动态加载逻辑**：
```javascript
// 监听服务器 ID 变化
watch(() => props.serverIds, (newIds) => {
  if (newIds && newIds.length > 0) {
    loadServerResources(newIds)
  }
}, { immediate: true })

// 调用后端 API 获取资源
async function loadServerResources(serverIds) {
  const response = await axios.post('/api/v1/servers/discovery/resources', {
    serverIds: serverIds
  })
  
  if (response.data.code === 0) {
    availableDisks.value = response.data.data.disks
    availableInterfaces.value = response.data.data.interfaces
  }
}
```

## 🎯 使用场景

### 场景 1：单服务器任务

1. 用户选择服务器 A（192.168.1.100）
2. 前端调用 API 获取服务器 A 的设备和网卡
3. 下拉框显示服务器 A 的真实设备（如 nvme0n1, nvme1n1）
4. 用户选择要采集的设备

### 场景 2：多服务器任务

1. 用户选择服务器 A 和服务器 B
2. 前端调用 API 获取两台服务器的设备汇总
3. 下拉框显示并集（sda, sdb, nvme0n1）
4. 用户选择要采集的设备（如选择 nvme0n1，则两台服务器都会采集）

### 场景 3：留空采集全部

1. 用户不选择任何设备/网卡
2. 采集时自动发现所有设备/网卡
3. 适用于设备配置不一致的场景

## 🔧 技术细节

### SSH 命令

**获取磁盘设备**：
```bash
lsblk -dn -o NAME,TYPE,SIZE | grep -E 'disk|lvm' | awk '{print $1}'
```

**获取网卡接口**：
```bash
ls /sys/class/net | grep -v '^lo$'
```

### 性能优化

1. **缓存机制**：获取一次后缓存，避免重复 SSH 连接
2. **超时控制**：SSH 命令超时 5 秒
3. **错误处理**：获取失败时不影响任务创建，使用空列表

## 📝 数据库设计

指标数据中的 `tags` 字段存储设备/网卡信息：

```sql
SELECT metric_name, tags FROM metrics WHERE task_id = 225;
```

**结果示例**：
| metric_name | tags |
|-------------|------|
| read_sectors_nvme0n1 | {"device":"nvme0n1"} |
| in_bytes_eth0 | {"interface":"eth0"} |

## 🎨 前端 UI


4. ✅ 支持多选
5. ✅ 支持"留空采集全部"
